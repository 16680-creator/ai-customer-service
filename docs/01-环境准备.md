## 2. JDK 17 安装与配置

### 2.1 Windows

1. 下载 JDK 17：
   - 推荐使用 [Adoptium Temurin 17](https://adoptium.net/temurin/releases/?version=17)
   - 选择 Windows x64 .msi 安装包

2. 运行安装程序，选择安装路径（如 `C:\Program Files\Java\jdk-17`）

3. 配置环境变量：
   ```powershell
   # 系统环境变量 → 新建
   JAVA_HOME = C:\Program Files\Java\jdk-17

   # 系统环境变量 → Path → 新增
   %JAVA_HOME%\bin
   ```

4. 验证安装：
   ```powershell
   java -version
   # 输出应类似：openjdk version "17.0.x" ...
   ```

### 2.2 macOS

```bash
# 使用 Homebrew 安装
brew install openjdk@17

# 配置环境变量（zsh）
echo 'export JAVA_HOME=$(/usr/libexec/java_home -v 17)' >> ~/.zshrc
echo 'export PATH=$JAVA_HOME/bin:$PATH' >> ~/.zshrc
source ~/.zshrc

# 验证
java -version
```

### 2.3 Linux (Ubuntu/Debian)

```bash
# 安装 Adoptium Temurin 17
sudo apt update
sudo apt install -y wget apt-transport-https gpg
wget -O - https://packages.adoptium.net/artifactory/api/gpg/key/public | sudo apt-key add -
echo "deb https://packages.adoptium.net/artifactory/deb $(lsb_release -cs) main" | sudo tee /etc/apt/sources.list.d/adoptium.list
sudo apt update
sudo apt install -y temurin-17-jdk

# 配置环境变量
echo 'export JAVA_HOME=/usr/lib/jvm/temurin-17-jdk-amd64' >> ~/.bashrc
echo 'export PATH=$JAVA_HOME/bin:$PATH' >> ~/.bashrc
source ~/.bashrc

# 验证
java -version
```

## 3. Maven 配置

### 3.1 安装 Maven

```bash
# macOS
brew install maven

# Linux
sudo apt install -y maven

# Windows：下载 https://maven.apache.org/download.cgi 解压后配置 MAVEN_HOME 和 Path
```

验证：
```bash
mvn -version
# Apache Maven 3.9.x
```

### 3.2 配置镜像源

编辑 Maven 配置文件 `~/.m2/settings.xml`（如不存在则新建）：

```xml
<?xml version="1.0" encoding="UTF-8"?>
<settings xmlns="http://maven.apache.org/SETTINGS/1.2.0"
          xmlns:xsi="http://www.w3.org/2001/XMLSchema-instance"
          xsi:schemaLocation="http://maven.apache.org/SETTINGS/1.2.0
          https://maven.apache.org/xsd/settings-1.2.0.xsd">

    <mirrors>
        <!-- 阿里云公共仓库 -->
        <mirror>
            <id>aliyunmaven</id>
            <mirrorOf>central</mirrorOf>
            <name>阿里云公共仓库</name>
            <url>https://maven.aliyun.com/repository/public</url>
        </mirror>
        <!-- 阿里云 Spring 仓库（Spring AI / Spring Milestones 需要） -->
        <mirror>
            <id>aliyun-spring</id>
            <mirrorOf>spring-milestones,spring-snapshots</mirrorOf>
            <name>阿里云 Spring 仓库</name>
            <url>https://maven.aliyun.com/repository/spring</url>
        </mirror>
    </mirrors>

    <profiles>
        <profile>
            <id>jdk-17</id>
            <activation>
                <activeByDefault>true</activeByDefault>
                <jdk>17</jdk>
            </activation>
            <properties>
                <maven.compiler.source>17</maven.compiler.source>
                <maven.compiler.target>17</maven.compiler.target>
                <maven.compiler.compilerVersion>17</maven.compiler.compilerVersion>
            </properties>
        </profile>
    </profiles>
</settings>
```

> **注意**：项目 `pom.xml` 中已配置了 Spring Milestones 和 Spring Snapshots 仓库，用于拉取 Spring AI 依赖。如果阿里云镜像同步延迟，可临时注释镜像配置直接从 Spring 仓库下载。

## 4. Docker 安装

### 4.1 Windows

1. 下载 [Docker Desktop for Windows](https://www.docker.com/products/docker-desktop/)
2. 安装并启动 Docker Desktop
3. 确认 WSL2 后端已启用（Docker Desktop → Settings → General → Use WSL 2 based engine）
4. 验证：
   ```powershell
   docker --version
   # Docker version 24.x.x
   docker compose version
   # Docker Compose version v2.x.x
   ```

### 4.2 macOS

```bash
brew install --cask docker
# 启动 Docker Desktop 应用
docker --version
docker compose version
```

### 4.3 Linux (Ubuntu)

```bash
# 安装 Docker Engine
sudo apt update
sudo apt install -y ca-certificates curl gnupg
sudo install -m 0755 -d /etc/apt/keyrings
curl -fsSL https://download.docker.com/linux/ubuntu/gpg | sudo gpg --dearmor -o /etc/apt/keyrings/docker.gpg
echo "deb [arch=$(dpkg --print-architecture) signed-by=/etc/apt/keyrings/docker.gpg] https://download.docker.com/linux/ubuntu $(lsb_release -cs) stable" | sudo tee /etc/apt/sources.list.d/docker.list > /dev/null
sudo apt update
sudo apt install -y docker-ce docker-ce-cli containerd.io docker-buildx-plugin docker-compose-plugin

# 将当前用户加入 docker 组
sudo usermod -aG docker $USER
newgrp docker

# 验证
docker --version
docker compose version
```

### 4.4 Docker 镜像加速（国内用户）

编辑 Docker Desktop → Settings → Docker Engine，添加：

```json
{
  "registry-mirrors": [
    "https://mirror.ccs.tencentyun.com",
    "https://docker.mirrors.ustc.edu.cn"
  ]
}
```

## 5. IDE 插件推荐

### 5.1 IntelliJ IDEA 插件

| 插件名称 | 用途 | 安装方式 |
|---------|------|---------|
| **Lombok** | 自动生成 getter/setter/constructor 等 | Settings → Plugins → 搜索 "Lombok" |
| **Spring Boot** | Spring Boot 开发支持、配置提示 | IDEA Ultimate 内置 |
| **MyBatis-Plus** | MyBatis-Plus 代码提示、XML 跳转 | Plugins → 搜索 "MyBatis-Plus" |
| **Spring AI** | Spring AI 代码提示（如有） | Plugins → 搜索 "Spring AI" |
| **Docker** | Dockerfile / Compose 语法支持 | IDEA Ultimate 内置 |
| **GitToolBox** | Git 增强工具 | Plugins → 搜索 "GitToolBox" |
| **MapStruct Support** | MapStruct 代码生成辅助 | Plugins → 搜索 "MapStruct" |

### 5.2 IDEA 项目配置

1. **启用 Annotation Processing**：
   - Settings → Build, Execution, Deployment → Compiler → Annotation Processors
   - 勾选 ✅ Enable annotation processing

2. **设置 JDK**：
   - File → Project Structure → Project → SDK → 选择 JDK 17
   - Project → Language Level → 选择 17

3. **设置 Maven**：
   - Settings → Build, Execution, Deployment → Build Tools → Maven
   - Maven home path → 选择本地 Maven 安装路径
   - User settings file → 指向上文配置的 `~/.m2/settings.xml`

## 6. 项目克隆与编译

### 6.1 克隆项目

```bash
# 克隆代码仓库
git clone <repository-url> ai-customer-service
cd ai-customer-service
```

### 6.2 项目结构

```
ai-customer-service/
├── pom.xml                    # 父 POM，管理所有模块和依赖版本
├── ai-cs-common/              # 公共模块（Result、BusinessException、JwtUtil）
├── ai-cs-gateway/             # API 网关（路由、鉴权、限流）
├── ai-cs-user/                # 用户服务（注册、登录、JWT鉴权）
├── ai-cs-knowledge/           # 知识库服务（文档管理、MinIO上传）
├── ai-cs-chat/                # AI 对话服务（Spring AI、RAG、流式对话）
├── ai-cs-search/              # 搜索服务（Elasticsearch 全文搜索）
├── ai-cs-message/             # 消息服务（RocketMQ 生产者/消费者）
├── ai-cs-notify/              # 通知服务（WebSocket 消息推送）
└── docs/                      # 项目文档
```

### 6.3 编译项目

```bash
# 全量编译（跳过测试）
mvn clean install -DskipTests

# 仅编译公共模块（其他模块依赖它）
mvn clean install -pl ai-cs-common -DskipTests

# 编译指定模块及其依赖
mvn clean install -pl ai-cs-user -am -DskipTests

# 查看依赖树
mvn dependency:tree -pl ai-cs-chat
```

### 6.4 常见编译问题

| 问题 | 原因 | 解决方案 |
|------|------|---------|
| `spring-ai-bom` 下载失败 | Spring AI 在 Milestones 仓库 | 确认 `settings.xml` 未屏蔽 Spring 仓库，或添加阿里云 Spring 镜像 |
| `找不到 ai-cs-common` | 未先安装公共模块 | 先执行 `mvn install -pl ai-cs-common` |
| `Lombok 不生效` | 未启用 Annotation Processing | IDEA → Settings → Compiler → Annotation Processors → 勾选 Enable |
| `MapStruct 生成类找不到` | 注解处理器顺序问题 | 确认 `pom.xml` 中 `annotationProcessorPaths` 配置了 Lombok 和 MapStruct |
