## 1. Docker Compose 一键启动

### 1.1 docker-compose.yml

在项目根目录创建 `deploy/docker-compose.yml`：

```yaml

      MYSQL_ROOT_PASSWORD: root

      - mysql-master-data:/var/lib/mysql
```

## 2. MySQL 主从复制配置

### 2.1 master.cnf

创建 `deploy/mysql/master.cnf`：

```ini
[mysqld]
# 服务器唯一ID
server-id=1
# 开启二进制日志
log-bin=mysql-bin
# 二进制日志格式
binlog-format=ROW
# 需要同步的数据库（多个用逗号分隔）
binlog-do-db=aics_user
binlog-do-db=aics_knowledge
binlog-do-db=nacos_config
# 不需要同步的数据库
binlog-ignore-db=mysql
binlog-ignore-db=information_schema
binlog-ignore-db=performance_schema
# 二进制日志过期时间（天）
expire-logs-days=7
# GTID 模式（可选，推荐开启）
gtid-mode=ON
enforce-gtid-consistency=ON
```

### 2.2 slave.cnf

创建 `deploy/mysql/slave.cnf`：

```ini
[mysqld]
# 服务器唯一ID（每个从库不同）
server-id=2
# 开启中继日志
relay-log=relay-bin
# 只读模式
read-only=1
# GTID 模式
gtid-mode=ON
enforce-gtid-consistency=ON
# 从库也开启二进制日志（级联复制时需要）
log-bin=mysql-bin
log-slave-updates=ON
```

> **注意**：`mysql-slave2` 的 `server-id` 应设为 `3`，需创建单独的 `slave2.cnf`。

### 2.3 初始化数据库

创建 `deploy/mysql/init/init.sql`：

```sql
-- 创建业务数据库
CREATE DATABASE IF NOT EXISTS aics_user DEFAULT CHARACTER SET utf8mb4 COLLATE utf8mb4_unicode_ci;
CREATE DATABASE IF NOT EXISTS aics_knowledge DEFAULT CHARACTER SET utf8mb4 COLLATE utf8mb4_unicode_ci;
CREATE DATABASE IF NOT EXISTS nacos_config DEFAULT CHARACTER SET utf8mb4 COLLATE utf8mb4_unicode_ci;

-- 用户表
USE aics_user;
CREATE TABLE IF NOT EXISTS t_user (
    id BIGINT PRIMARY KEY COMMENT '主键ID',
    username VARCHAR(50) NOT NULL UNIQUE COMMENT '用户名',
    password VARCHAR(100) NOT NULL COMMENT '密码',
    nickname VARCHAR(50) COMMENT '昵称',
    phone VARCHAR(20) COMMENT '手机号',
    email VARCHAR(100) COMMENT '邮箱',
    avatar VARCHAR(255) COMMENT '头像URL',
    status INT DEFAULT 1 COMMENT '状态：0-禁用 1-启用',
    role VARCHAR(20) DEFAULT 'user' COMMENT '角色标识',
    create_time DATETIME DEFAULT CURRENT_TIMESTAMP COMMENT '创建时间',
    update_time DATETIME DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP COMMENT '更新时间',
    deleted INT DEFAULT 0 COMMENT '逻辑删除：0-未删除 1-已删除'
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COMMENT='用户表';

-- 知识文档表
USE aics_knowledge;
CREATE TABLE IF NOT EXISTS t_knowledge_document (
    id BIGINT PRIMARY KEY COMMENT '主键ID',
    title VARCHAR(200) NOT NULL COMMENT '文档标题',
    content LONGTEXT COMMENT '文档内容',
    doc_type VARCHAR(20) COMMENT '文档类型：pdf/docx/txt/markdown/html',
    source_url VARCHAR(500) COMMENT '文档来源URL',
    summary VARCHAR(500) COMMENT '文档摘要',
    tags VARCHAR(500) COMMENT '标签（逗号分隔）',
    status INT DEFAULT 0 COMMENT '状态：0-待处理 1-已索引 2-索引失败',
    create_by BIGINT COMMENT '创建人ID',
    create_time DATETIME DEFAULT CURRENT_TIMESTAMP COMMENT '创建时间',
    update_time DATETIME DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP COMMENT '更新时间',
    deleted INT DEFAULT 0 COMMENT '逻辑删除：0-未删除 1-已删除'
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COMMENT='知识文档表';
```

### 2.4 配置主从同步

```bash
# 1. 进入主库容器
docker exec -it aics-mysql-master mysql -uroot -proot

# 2. 创建复制用户
CREATE USER 'repl'@'%' IDENTIFIED WITH mysql_native_password BY 'repl_password';
GRANT REPLICATION SLAVE ON *.* TO 'repl'@'%';
FLUSH PRIVILEGES;

# 3. 查看主库状态（记录 File 和 Position）
SHOW MASTER STATUS;

# 4. 进入从库容器
docker exec -it aics-mysql-slave1 mysql -uroot -proot

# 5. 配置主从复制
CHANGE MASTER TO
    MASTER_HOST='mysql-master',
    MASTER_PORT=3306,
    MASTER_USER='repl',
    MASTER_PASSWORD='repl_password',
    MASTER_LOG_FILE='mysql-bin.000001',
    MASTER_LOG_POS=0;

# 6. 启动从库复制
START SLAVE;

# 7. 验证主从同步状态
SHOW SLAVE STATUS\G
```

### 2.5 验证主从同步

```bash
# 在主库创建测试数据
docker exec -it aics-mysql-master mysql -uroot -proot -e \
  "USE aics_user; INSERT INTO t_user (id, username, password, nickname) VALUES (1, 'test', 'test123', 'TestUser');"

# 在从库查询验证
docker exec -it aics-mysql-slave1 mysql -uroot -proot -e \
  "USE aics_user; SELECT * FROM t_user WHERE username='test';"

# 关键验证指标
# SHOW SLAVE STATUS\G 输出中：
#   Slave_IO_Running: Yes
#   Slave_SQL_Running: Yes
#   Seconds_Behind_Master: 0
```

## 3. Redis 集群配置

### 3.1 节点配置文件

创建 `deploy/redis/node1.conf`（node2、node3 类似，修改端口和路径）：

```conf
# deploy/redis/node1.conf
port 6379
cluster-enabled yes
cluster-config-file nodes.conf
cluster-node-timeout 5000
appendonly yes
bind 0.0.0.0
protected-mode no

# 集群总线端口（默认为 port + 10000）
cluster-announce-ip 127.0.0.1
cluster-announce-port 6379
cluster-announce-bus-port 16379
```

`node2.conf` 和 `node3.conf` 分别修改端口为 `6380/16380` 和 `6381/16381`。

### 3.2 创建集群

```bash
# 等待三个节点启动完成后执行
docker exec -it aics-redis-node1 redis-cli --cluster create \
  127.0.0.1:6379 \
  127.0.0.1:6380 \
  127.0.0.1:6381 \
  --cluster-replicas 0

# 输入 yes 确认集群创建
```

### 3.3 验证集群

```bash
# 查看集群信息
docker exec -it aics-redis-node1 redis-cli cluster info

# 查看集群节点
docker exec -it aics-redis-node1 redis-cli cluster nodes

# 测试写入
docker exec -it aics-redis-node1 redis-cli set test_key "hello_aics"

# 测试读取
docker exec -it aics-redis-node2 redis-cli get test_key
```

## 8. Seata 配置

### 8.1 application.yml

创建 `deploy/seata/application.yml`：

```yaml
server:
  port: 7091

spring:
  application:
    name: seata-server

logging:
  level:
    io.seata: info

seata:
  config:
    type: nacos
    nacos:
      server-addr: nacos:8848
      namespace: aics
      group: SEATA_GROUP
      data-id: seataServer.properties
  registry:
    type: nacos
    nacos:
      application: seata-server
      server-addr: nacos:8848
      namespace: aics
      group: SEATA_GROUP
      cluster: default
  store:
    mode: file
  security:
    secretKey: SeataSecretKey
    tokenValidityInMilliseconds: 1800000
    ignore:
      urls: /,/**/*.css,/**/*.js,/**/*.html,/**/*.map,/**/*.svg,/**/*.png,/**/*.ico,/console-fe/public/**,/api/v1/auth/login
```

### 8.2 在 Nacos 中创建 Seata 配置

在 Nacos 控制台 → `aics` 命名空间 → 新建配置：

- **Data ID**：`seataServer.properties`
- **Group**：`SEATA_GROUP`
- **内容**：

```properties
# 存储模式
store.mode=file
store.file.dir=sessionStore

# 事务分组
service.vgroupMapping.aics_tx_group=default
service.default.grouplist=seata-server:8091
```

### 8.3 验证 Seata 注册


## 9. 常见问题排查

### 9.1 容器启动失败

```bash
# 查看容器日志
docker compose logs <service-name>

# 查看容器退出码
docker inspect <container-name> --format='{{.State.ExitCode}}'

# 常见退出码：
# 137 - OOM（内存不足），需增加 Docker 内存限制
# 1   - 应用错误，查看日志定位
```

### 9.2 MySQL 主从不同步

```bash
# 在从库检查状态
docker exec -it aics-mysql-slave1 mysql -uroot -proot -e "SHOW SLAVE STATUS\G"

# 常见问题：
# Last_IO_Error: 连接主库失败 → 检查网络和复制用户
# Last_SQL_Error: SQL 执行错误 → 检查数据一致性

# 重新配置主从（在从库执行）
STOP SLAVE;
RESET SLAVE ALL;
CHANGE MASTER TO MASTER_HOST='mysql-master', MASTER_PORT=3306, MASTER_USER='repl', MASTER_PASSWORD='repl_password', MASTER_AUTO_POSITION=1;
START SLAVE;
```

### 9.3 Nacos 启动报数据库错误

```bash
# 确认 MySQL 中已创建 nacos_config 数据库
docker exec -it aics-mysql-master mysql -uroot -proot -e "SHOW DATABASES LIKE 'nacos_config';"
```

### 9.4 Elasticsearch 启动失败

```bash
# 常见原因：vm.max_map_count 太低
# Linux 临时修改
sudo sysctl -w vm.max_map_count=262144

# 永久修改
echo "vm.max_map_count=262144" | sudo tee -a /etc/sysctl.conf
sudo sysctl -p

# macOS (Docker Desktop) 通常不需要修改
# Windows (WSL2) 在 PowerShell 中执行：
wsl -d docker-desktop -e sysctl -w vm.max_map_count=262144
```

### 9.6 Redis 集群创建失败

```bash
# 确保所有节点都已启动
docker compose ps redis-node1 redis-node2 redis-node3

# 确保节点间网络互通
docker exec -it aics-redis-node1 redis-cli -p 6380 ping
docker exec -it aics-redis-node1 redis-cli -p 6381 ping

# 如果集群创建失败，重置节点
docker exec -it aics-redis-node1 redis-cli flushall
docker exec -it aics-redis-node1 redis-cli cluster reset
docker exec -it aics-redis-node2 redis-cli flushall
docker exec -it aics-redis-node2 redis-cli cluster reset
docker exec -it aics-redis-node3 redis-cli flushall
docker exec -it aics-redis-node3 redis-cli cluster reset
# 然后重新创建集群
```
