
**Maven 配置：**

编辑 `~/.m2/settings.xml`（Windows 路径：`C:\Users\用户名\.m2\settings.xml`）：

```xml
<?xml version="1.0" encoding="UTF-8"?>
<settings>
    <mirrors>
        <mirror>
            <id>aliyun</id>
            <mirrorOf>central</mirrorOf>
            <name>阿里云镜像</name>
            <url>https://maven.aliyun.com/repository/public</url>
        </mirror>
    </mirrors>
</settings>
```

**Docker 资源分配：**

Docker Desktop → Settings → Resources：
- CPU: 6 核
- Memory: 12 GB
- Disk: 60 GB


## 第四部分：数据库初始化与主从复制

### 4.1 数据库初始化

`docker-compose.yml` 启动时会自动执行 `deploy/mysql/init.sql`，创建以下数据库：

| 数据库 | 用途 | 表清单 |
|--------|------|--------|
| `user_db` | 用户中心 | sys_user, sys_role, sys_user_role |

| `chat_db` | 对话记录 | chat_session, chat_message, chat_feedback |
| `nacos_config` | Nacos 配置存储 | config_info, users, roles 等 11 张表 |

### 4.2 初始账号

| 账号 | 密码 | 角色 |
|------|------|------|
| admin | admin123 | 管理员 |
| nacos | nacos | Nacos 管理员 |

### 4.3 配置 MySQL 主从复制

**在主库创建复制用户：**

```bash
docker exec -it aics-mysql-master mysql -uroot -proot
```

```sql
CREATE USER 'repl'@'%' IDENTIFIED WITH mysql_native_password BY 'repl_password';
GRANT REPLICATION SLAVE ON *.* TO 'repl'@'%';
FLUSH PRIVILEGES;
SHOW MASTER STATUS;
-- 记录 File 和 Position 的值
```

**在从库1配置复制：**

```bash
docker exec -it aics-mysql-slave1 mysql -uroot -proot
```

```sql
CHANGE MASTER TO
    MASTER_HOST='mysql-master',
    MASTER_PORT=3306,
    MASTER_USER='repl',
    MASTER_PASSWORD='repl_password',
    MASTER_AUTO_POSITION=1;
START SLAVE;
SHOW SLAVE STATUS\G
-- 确认 Slave_IO_Running: Yes, Slave_SQL_Running: Yes
```

**在从库2配置复制：**

```bash
docker exec -it aics-mysql-slave2 mysql -uroot -proot
```

```sql
CHANGE MASTER TO
    MASTER_HOST='mysql-master',
    MASTER_PORT=3306,
    MASTER_USER='repl',
    MASTER_PASSWORD='repl_password',
    MASTER_AUTO_POSITION=1;
START SLAVE;
SHOW SLAVE STATUS\G
```

### 4.4 验证主从同步

### 6.3 导入服务配置

在 `aics` 命名空间下创建以下配置：

**配置 1：`ai-cs-user.yml`（用户服务配置）**

```yaml
spring:
  datasource:
    url: jdbc:mysql://mysql-master:3306/user_db?useUnicode=true&characterEncoding=utf-8&useSSL=false&serverTimezone=Asia/Shanghai&allowPublicKeyRetrieval=true
    username: root
    password: root
jwt:
  secret: aics-platform-jwt-secret-key-must-be-at-least-256-bits-long-for-hs256
  expiration: 86400000
```

**配置 2：`ai-cs-knowledge.yml`（知识库服务配置）**

```yaml
spring:
  datasource:
    url: jdbc:mysql://mysql-master:3306/knowledge_db?useUnicode=true&characterEncoding=utf-8&useSSL=false&serverTimezone=Asia/Shanghai&allowPublicKeyRetrieval=true
    username: root
    password: root
minio:
  endpoint: http://minio:9000
  access-key: minioadmin
  secret-key: minioadmin
  bucket: aics-knowledge
```

**配置 3：`ai-cs-chat.yml`（AI 对话服务配置）**

```yaml
spring:
  ai:
    openai:
      api-key: ${OPENAI_API_KEY:demo-key}
      base-url: ${OPENAI_BASE_URL:http://localhost:11434/v1}
      chat:
        options:
          model: ${OPENAI_MODEL:qwen2.5:7b}
          temperature: 0.7
          max-tokens: 2048
```

**配置 4：`ai-cs-search.yml`（搜索服务配置）**

```yaml
spring:
  elasticsearch:
    uris: http://elasticsearch:9200
```

**配置 5：`ai-cs-message.yml`（消息服务配置）**

```yaml
spring:
  datasource:
    url: jdbc:mysql://mysql-master:3306/chat_db?useUnicode=true&characterEncoding=utf-8&useSSL=false&serverTimezone=Asia/Shanghai&allowPublicKeyRetrieval=true
    username: root
    password: root
rocketmq:
  name-server: rocketmq-namesrv:9876
  producer:
    group: chat-producer-group
  consumer:
    group: chat-consumer-group
```

## 第七部分：微服务启动

### 7.1 编译项目

```bash
cd ai-customer-service

# 编译所有模块（跳过测试）
mvn clean package -DskipTests

# 如果编译失败，先编译公共模块
mvn clean install -pl ai-cs-common -DskipTests
mvn clean package -DskipTests
```

### 7.2 启动顺序

**推荐启动顺序：**

1. 中间件（已通过 docker-compose 启动，见第三部分）
2. 微服务（按依赖关系依次启动）：

```bash
# 启动用户服务（端口 8081）
mvn spring-boot:run -pl ai-cs-user -DskipTests &

# 启动知识库服务（端口 8082）
mvn spring-boot:run -pl ai-cs-knowledge -DskipTests &

# 启动 AI 对话服务（端口 8083）
mvn spring-boot:run -pl ai-cs-chat -DskipTests &

# 启动搜索服务（端口 8084）
mvn spring-boot:run -pl ai-cs-search -DskipTests &

# 启动消息服务（端口 8085）
mvn spring-boot:run -pl ai-cs-message -DskipTests &

# 启动通知服务（端口 8086）
mvn spring-boot:run -pl ai-cs-notify -DskipTests &

# 最后启动网关（端口 8080）
mvn spring-boot:run -pl ai-cs-gateway -DskipTests &
```

### 7.3 验证服务注册

```bash

# ai-cs-gateway, ai-cs-user, ai-cs-knowledge, ai-cs-chat,

```

```bash
# 安装 Ollama
# Windows: https://ollama.com/download/windows

# 下载模型

# 验证模型
ollama list
```

修改 `ai-cs-chat` 的 `application.yml` 或 Nacos 配置：

```yaml
spring:
  ai:
    openai:
      api-key: demo-key
      base-url: http://localhost:11434/v1
      chat:
        options:
          model: qwen2.5:7b
```

```yaml
spring:
  ai:
    openai:
      api-key: sk-your-api-key
      base-url: https://api.openai.com
      chat:
        options:
          model: gpt-3.5-turbo
```

设置环境变量：

```bash
export OPENAI_API_KEY=sk-your-api-key
```

## 第十三部分：故障排查

### 13.1 Docker 相关

**MySQL 容器启动失败：**

```bash
docker logs aics-mysql-master
# 常见原因：端口冲突（3306被占用）
# 解决：修改 docker-compose.yml 中 MySQL Master 端口映射
```

**Nacos 启动报数据库错误：**

```bash
# 确认 MySQL 中的 nacos_config 数据库已创建
docker exec aics-mysql-master mysql -uroot -proot -e "SHOW TABLES FROM nacos_config;"
# 如果为空，手动执行 init.sql 中的 nacos 部分
```

**Elasticsearch 启动失败（vm.max_map_count）：**

```bash
# Windows (WSL2)
wsl -d docker-desktop -e sysctl -w vm.max_map_count=262144

# Linux
sudo sysctl -w vm.max_map_count=262144
echo "vm.max_map_count=262144" | sudo tee -a /etc/sysctl.conf
```

### 13.2 微服务相关

**服务无法注册到 Nacos：**

```bash
# 检查 Nacos 配置
# 1. 确认 application.yml 中 nacos 地址正确
# 2. 确认命名空间 ID 为 "aics"
# 3. 检查网络：docker network inspect aics-network
```

**数据库连接失败：**

```bash
# 检查数据库连接配置
# 在 Docker 网络中，使用容器名作为 host
# 正确：mysql-master:3306
# 错误：localhost:3306（容器内 localhost 是容器自身）
```

**AI 对话超时：**

```bash
# Ollama 模型是否下载
ollama list

# 检查 Ollama 服务状态
curl http://localhost:11434/api/tags
```
